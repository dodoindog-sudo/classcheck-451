# ระบบเช็คชื่อเข้าเรียน — 01003451 สรีรวิทยาการผลิตพืชไร่

เว็บไซต์เช็คชื่อเข้าเรียน (HTML + JavaScript ล้วน รันบน GitHub Pages) สำหรับรายวิชา
Physiology of Field Crop Production ภาคต้น ปีการศึกษา 2569 คณะเกษตร มหาวิทยาลัยเกษตรศาสตร์

**คุณสมบัติ**
- เปิดให้เช็คชื่อได้เฉพาะวันที่มีเรียนตามตาราง (course syllabus) และเฉพาะช่วงเวลาที่กำหนดเท่านั้น
- ตรวจสอบตำแหน่ง GPS ของผู้ใช้ ต้องอยู่ภายในรัศมีที่กำหนดรอบคณะเกษตร มก. บางเขน จึงจะกรอกฟอร์มได้
- 1 อุปกรณ์ เช็คชื่อได้ 1 ครั้ง ต่อ 1 วันเรียน (ตรวจสอบซ้ำทั้งฝั่ง browser และฝั่งเซิร์ฟเวอร์)
- ตรวจรหัสนิสิตกับทะเบียนรายชื่อจริง กันคนนอกวิชากรอกฟอร์ม
- บันทึกผลลง Google Sheets ในบัญชี Google Drive ของอาจารย์เอง
- มีหน้า **admin.html** สรุปผลการเข้าเรียนของนิสิตทุกคนแบบตาราง (ล็อกด้วยรหัสผ่าน) + ดาวน์โหลด CSV

**ข้อมูลนิสิต (รหัส/ชื่อ-นามสกุล) จะไม่ถูกเก็บไว้ในโค้ดหรือ GitHub เด็ดขาด** เพราะ repo นี้เป็น public
รายชื่อจะถูกเก็บเฉพาะใน Google Sheet ส่วนตัวของอาจารย์เท่านั้น (ดูขั้นตอนที่ 2)

---

## ภาพรวมระบบ

```
เว็บ (GitHub Pages, HTML+JS)  --POST-->  Google Apps Script (Web App)  -->  Google Sheet ในไดรฟ์อาจารย์
```

ฝั่งเว็บไม่มี backend ของตัวเอง จึงต้องใช้ Google Apps Script เป็นตัวกลางรับข้อมูลแล้วบันทึกลง Google Sheets
(Sheet อยู่ใน Google Drive ของอาจารย์ ตรงตามที่ต้องการ "เก็บข้อมูลไว้ใน Google Drive")

---

## ขั้นตอนที่ 1: สร้าง Google Sheet + Apps Script

1. ไปที่ [sheets.google.com](https://sheets.google.com) สร้างสเปรดชีตใหม่ ตั้งชื่อเช่น `เช็คชื่อ 01003451`
2. สร้างชีตชื่อ **Roster** (พิมพ์ชื่อชีตให้ตรงตัวพิมพ์ใหญ่เล็ก) ใส่หัวตาราง:
   - A1 = `รหัสนิสิต`, B1 = `ชื่อ-นามสกุล`
   - แล้ว**วางรายชื่อนิสิตทั้งหมด** (ผมเตรียมไฟล์ CSV ไว้ให้แยกต่างหาก ไม่ได้ใส่ไว้ในโปรเจกต์นี้เพราะเป็นข้อมูลส่วนบุคคล — ดูหมายเหตุท้ายไฟล์นี้)
3. สร้างชีตชื่อ **Attendance** เว้นว่างไว้ (ระบบจะสร้างหัวตารางให้อัตโนมัติเมื่อมีการเช็คชื่อครั้งแรก)
4. เมนู **ส่วนขยาย (Extensions) > Apps Script**
5. ลบโค้ดเดิมทั้งหมด แล้ว**คัดลอกเนื้อหาไฟล์ [`apps-script/Code.gs`](apps-script/Code.gs) ในโปรเจกต์นี้ไปวาง**
6. (ไม่บังคับ) แก้ค่า `API_TOKEN` ในบรรทัดแรก ๆ ของโค้ดให้เป็นค่าที่ตั้งเอง แล้วจำไว้ไปใส่ในขั้นตอนที่ 3
   **ต้องแก้** ค่า `ADMIN_TOKEN` เป็นรหัสผ่านของอาจารย์เอง (คนละค่ากับ `API_TOKEN`) ใช้สำหรับล็อกอินหน้า admin.html — ห้ามใช้รหัสผ่านซ้ำกับบัญชีอื่น
7. กด **Deploy > New deployment**
   - Select type: **Web app**
   - Execute as: **Me (บัญชีของอาจารย์)**
   - Who has access: **Anyone**
   - กด Deploy แล้วอนุญาตสิทธิ์ (Authorize access) ตามที่ Google ขอ
8. คัดลอก **Web app URL** ที่ได้ (รูปแบบ `https://script.google.com/macros/s/xxxxx/exec`) เก็บไว้ใช้ในขั้นตอนถัดไป

> หมายเหตุ: ทุกครั้งที่แก้โค้ด Code.gs ต้องกด Deploy > Manage deployments > แก้ไข (ไอคอนดินสอ) > Version: New version > Deploy ใหม่ URL เดิมจะยังใช้ได้

## ขั้นตอนที่ 3: ตั้งค่าเว็บไซต์ (js/config.js)

เปิดไฟล์ [`js/config.js`](js/config.js) แก้ 2 บรรทัดนี้:

```js
APPS_SCRIPT_URL: "วาง Web app URL จากขั้นตอนที่ 1",
API_TOKEN: "ต้องตรงกับ API_TOKEN ใน Code.gs",
```

ค่าที่ตั้งไว้ล่วงหน้าอื่น ๆ (ปรับได้ตามต้องการ):

| ค่า | ความหมาย | ค่าเริ่มต้น |
|---|---|---|
| `GEOFENCE.lat/lng` | จุดศูนย์กลางคณะเกษตร มก. บางเขน | 13.8498543, 100.5710172 |
| `GEOFENCE.radiusMeters` | รัศมีที่อนุญาตให้เช็คชื่อ | 450 เมตร |
| `CHECKIN_WINDOW` | เปิด/ปิดรับเช็คชื่อก่อน/หลังเวลาเรียน | เปิดก่อนเรียน 30 นาที ปิดหลังเลิกเรียน 30 นาที |
| `SESSIONS` | รายการวันเรียนตาม course syllabus | 14 วันเรียน (ไม่รวมสัปดาห์งดเรียน/สอบ) |

**สำคัญ:** ถ้าแก้ `SESSIONS` หรือเวลาเรียนในไฟล์นี้ ต้องแก้ค่า `SESSIONS`/`CLASS_START`/`CLASS_END`
ใน `apps-script/Code.gs` ให้ตรงกันด้วย (ฝั่งเซิร์ฟเวอร์ตรวจซ้ำเพื่อกันการปลอมวันที่/เวลาจากเบราว์เซอร์)

ตำแหน่งพิกัดที่ตั้งไว้เป็นตำแหน่งหมุด "คณะเกษตร มหาวิทยาลัยเกษตรศาสตร์" จาก Google Maps
แนะนำให้อาจารย์เปิด Google Maps ตรวจสอบพิกัดจริงของอาคารวชิรานุสรณ์อีกครั้งก่อนใช้งานจริง
และปรับรัศมีให้ครอบคลุมพื้นที่ที่ต้องการ (ไม่กว้างเกินจนคลุมพื้นที่นอกคณะ)

## ขั้นตอนที่ 4: สร้าง GitHub repo และเปิด GitHub Pages

1. สมัครบัญชีที่ [github.com](https://github.com) (ถ้ายังไม่มี)
2. สร้าง repository ใหม่ ตั้งเป็น **Public** เช่นชื่อ `crop451-attendance` (ห้ามติ๊ก README/gitignore ใด ๆ เพิ่ม เพราะโปรเจกต์นี้มีไฟล์ครบแล้ว)
3. ในเครื่องที่มีโปรเจกต์นี้ รันคำสั่ง (แทน `<username>` และ `<repo>` ด้วยของจริง):

```bash
git init
git add .
git commit -m "Initial attendance system for 01003451"
git branch -M main
git remote add origin https://github.com/<username>/<repo>.git
git push -u origin main
```

(ระบบจะเปิดหน้าต่างให้ล็อกอิน GitHub ผ่านเบราว์เซอร์ในการ push ครั้งแรก)

4. ใน GitHub repo ไปที่ **Settings > Pages**
   - Source: **Deploy from a branch**
   - Branch: **main** / โฟลเดอร์ **/(root)**
   - กด Save
5. รอ 1-2 นาที จะได้ลิงก์เว็บไซต์รูปแบบ `https://<username>.github.io/<repo>/`
   นำลิงก์นี้ไปแจกให้นิสิตใช้เช็คชื่อ (เช่น แปะใน Google Classroom)

## หน้าสรุปผลสำหรับอาจารย์ (admin.html)

เปิดที่ `https://<username>.github.io/<repo>/admin.html` (มีลิงก์เล็ก ๆ ท้ายหน้าเช็คชื่อหลักด้วย)
กรอกรหัสผ่าน `ADMIN_TOKEN` ที่ตั้งไว้ใน Code.gs จะเห็นตารางสรุปว่านิสิตแต่ละคนมาเรียนวันไหนบ้าง
กี่ % ของทั้งหมด (ไฮไลต์สีแดงถ้าต่ำกว่า 80% ตามเกณฑ์ในซิลาบัส) พร้อมปุ่มดาวน์โหลดเป็น CSV

รหัสผ่านจะไม่ถูกบันทึกไว้ในเบราว์เซอร์ ต้องกรอกใหม่ทุกครั้งที่เปิดหน้านี้ และอย่าลืมว่านี่เป็นเว็บ static
รหัสผ่านนี้เป็นการกันคนทั่วไปเข้าถึงโดยไม่ตั้งใจเท่านั้น ไม่ใช่การยืนยันตัวตนระดับสูง (ดูหัวข้อข้อจำกัดด้านล่าง)

## ทดสอบก่อนใช้งานจริง

- เปิดลิงก์เว็บไซต์ในมือถือขณะอยู่ในพื้นที่คณะเกษตร ทดลองกดตรวจสอบ GPS และกรอกรหัสนิสิตที่มีในทะเบียนดูก่อน
- ทดลองกรอกรหัสนิสิตซ้ำ หรือใช้เครื่องเดิมเช็คชื่อซ้ำ ต้องขึ้นข้อความแจ้งว่าเช็คชื่อไปแล้ว
- ทดลองนอกเวลาเรียน/นอกวันเรียน ต้องขึ้นข้อความว่าไม่อยู่ในช่วงเวลา/ไม่ใช่วันเรียน
- ตรวจผลใน Google Sheet ชีต **Attendance** ว่าบันทึกข้อมูลถูกต้องครบถ้วน

## ข้อจำกัดที่ควรทราบ

- การตรวจ GPS ทำผ่านเบราว์เซอร์ ผู้ใช้ที่ตั้งใจปลอมพิกัด (mock location/แก้ผ่าน dev tools) อาจหลบเลี่ยงได้
  ระบบนี้เป็นการป้องกันระดับ "กันคนทั่วไปทำผิดโดยไม่ตั้งใจ / เพิ่มความน่าเชื่อถือของข้อมูล" ไม่ใช่การยืนยันตัวตนระดับสูง
  หากต้องการความเข้มงวดสูงสุด แนะนำใช้ควบคู่กับการเดินตรวจสอบหน้าห้องเป็นระยะ
- การจำกัด "1 อุปกรณ์ต่อ 1 ครั้ง" อ้างอิงจากรหัสอุปกรณ์ที่เก็บใน localStorage ของเบราว์เซอร์
  หากนิสิตล้างข้อมูลเบราว์เซอร์ หรือใช้เบราว์เซอร์อื่น/โหมดไม่ระบุตัวตน อาจเช็คชื่อซ้ำได้ในเชิงเทคนิค
  แต่ระบบยังกันไม่ให้ **รหัสนิสิตเดียวกัน** เช็คชื่อซ้ำในวันเดียวกันได้เสมอ (ตรวจที่ฝั่งเซิร์ฟเวอร์)
- หน้า admin.html ป้องกันด้วยรหัสผ่าน (`ADMIN_TOKEN`) เท่านั้น ไม่ใช่ระบบล็อกอินจริง เพราะเว็บนี้เป็น static site
  ไม่ควรใช้รหัสผ่านซ้ำกับบัญชีอื่น และควรเปลี่ยนรหัสเป็นระยะหากสงสัยว่ารั่วไหล
